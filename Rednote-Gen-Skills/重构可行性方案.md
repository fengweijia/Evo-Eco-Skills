## 一、需求理解确认

### 1.1 核心功能流程（7大模块）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           rednote002 工作流                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ① 账号配置                                                                  │
│  输入: 账号定位、目标受众、运营目标、调性风格、内容制作构想                      │
│  → 输出: 结构化账号画像                                                      │
│         ↓                                                                   │
│  ② 对标账号搜集                                                              │
│  输入: 内容制作方向                                                          │
│  → 输出: 短视频平台（抖音/快手/B站）对标账号列表                              │
│         ↓                                                                   │
│  ③ 账号设计（新增！）                                                        │
│  分析: 对标账号特征 + 账号定位                                               │
│  → 输出: 差异化账号名称、介绍、背景图、logo、人设                             │
│         ↓                                                                   │
│  ④ 内容拆解分析                                                              │
│  输入: 对标账号内容                                                          │
│  → 输出: 爆款类型、发布频率、互动特征                                        │
│         ↓                                                                   │
│  ⑤ 主题规划                                                                  │
│  输入: 账号定位 + 热点                                                       │
│  → 输出: 3-5个推荐主题（用户确认）                                           │
│         ↓                                                                   │
│  ⑥ 笔记生成                                                                  │
│  用户确认主题后                                                              │
│  → 输出: 标题、正文、封面图、标签、发布时间                                   │
│         ↓                                                                   │
│  ⑦ 数据监控                                                                  │
│  持续追踪                                                                    │
│  → 输出: 优化建议                                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 与原版rednote002的差异

| 功能 | 原版 | 重构版 | 说明 |
|------|------|--------|------|
| 账号设计 | ❌ 无 | ✅ 新增 | 差异化名称/介绍/logo/人设 |
| 对标数据分析 | 基础 | 深入 | 增加发布频率、互动特征分析 |
| 热点追踪 | 模拟 | 真实 | 接入热点API |
| 封面图生成 | 建议 | AI生成 | DALL-E/通义万相 |

## 二、GitHub开源项目调研

### 2.1 核心数据源（必须）

| 项目 | Stars | 用途 | 集成难度 |
|------|-------|------|----------|
| **TikHub-API-Python-SDK** | ⭐586 | 多平台API（抖音/小红书/快手/B站） | ⭐⭐ 中等 |
| **JoeanAmier/XHS-Downloader** | ⭐10k+ | 小红书数据采集 | ⭐⭐⭐ 较难 |
| **Autoxhs** | ⭐284 | AI自动生成+发布小红书 | ⭐⭐ 中等 |
| **Agent-Reach** | ⭐5k+ | 社媒数据采集（Twitter/Reddit/B站/小红书） | ⭐⭐ 中等 |

### 2.2 账号分析（必须）

| 项目 | Stars | 用途 | 集成难度 |
|------|-------|------|----------|
| **social-push** | ⭐238 | AI社交媒体自动化+自我进化 | ⭐⭐ 适中 |
| **apify-mcp-server** | ⭐854 | 社交媒体数据提取 | ⭐⭐ 中等 |

### 2.3 AI内容生成

| 项目 | Stars | 用途 | 集成难度 |
|------|-------|------|----------|
| **Xiaohongshu-Content-Generator** | ⭐26 | 将博客文章转为小红书风格 | ⭐ 简单 |
| **DALL-E / Stable Diffusion** | - | 封面图生成 | ⭐ 简单 |

### 2.4 推荐组合方案

#### 方案A：完全自建（推荐）
```
数据来源: TikHub SDK → 爬取对标账号数据
数据分析: 自建分析模块
内容生成: OpenAI API / 阿里通义万相
封面生成: DALL-E 3 / Stable Diffusion
```

#### 方案B：集成现有工具
```
数据来源: Agent-Reach (多平台统一)
内容生成: Autoxhs (已集成OpenAI)
封面生成: 单独接入AI绘图API
```

## 三、技术架构设计

### 3.1 多Agent架构

```
                        ┌─────────────────┐
                        │   协调Agent     │
                        │  (Orchestrator) │
                        └────────┬────────┘
                                 │
    ┌────────────────────────────┼────────────────────────────┐
    │                            │                            │
    ▼                            ▼                            ▼
┌───────────────┐      ┌─────────────────┐      ┌─────────────────┐
│ 账号配置Agent │      │  对标分析Agent   │      │  账号设计Agent  │
│ (Profiler)    │      │  (Analyzer)     │      │  (Designer)     │
└───────┬───────┘      └────────┬────────┘      └────────┬────────┘
        │                       │                        │
        └───────────────────────┼────────────────────────┘
                                ▼
                    ┌─────────────────────┐
                    │   内容生成Agent     │
                    │   (Generator)       │
                    └──────────┬──────────┘
                               │
                    ┌──────────┴──────────┐
                    ▼                     ▼
            ┌───────────────┐    ┌───────────────┐
            │ 笔记生成Agent │    │ 数据监控Agent │
            │ (NoteGen)     │    │ (Monitor)     │
            └───────────────┘    └───────────────┘
```

### 3.2 技术栈

- **Skills框架**: 按skill-creator规范构建
- **后端**: Node.js Serverless / Python
- **AI**: OpenAI API / 阿里通义万相
- **数据源**: TikHub SDK / Agent-Reach
- **部署**: Docker / 阿里云函数计算
